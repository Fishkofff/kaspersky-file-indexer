# Библиотека

Многопоточная библиотека на чистой Java для индексации текстовых файлов и быстрого поиска по словам. Библиотека умеет рекурсивно отслеживать изменения в файловой системе (создание, удаление и редактирование файлов) и обновлять индекс «на лету» без блокировки читающих потоков.

##  Архитектура проекта

Проект спроектирован по модульной архитектуре с использованием Gradle Multi-project builds. Это гарантирует строгую изоляцию логики:

1. **`indexer.core` (Библиотека):** Ядро системы. Не имеет зависимостей (кроме JUnit для тестов). Содержит всю логику индексации, токенизации и многопоточной работы. Предоставляет чистый API через фасад `FileIndexer`.
2. **`indexer.cli` (Приложение):** Тонкая обертка над библиотекой, предоставляющая консольный интерфейс (REPL) для взаимодействия с пользователем.

##  Технические составляющие

* **Lock-Free Индекс (`InvertedIndex`):**
  Вместо глобальных блокировок (`synchronized`) на весь индекс, используется `ConcurrentHashMap` и его атомарные операции `compute()`. Это позволяет десяткам потоков обновлять разные файлы абсолютно параллельно. Вспомогательная мапа (`fileWords`) позволяет мгновенно удалять из индекса слова, которые исчезли из файла после редактирования ($O(K)$ вместо $O(N)$).
* **Асинхронный WatchService (`DirectoryWatcher`):**
  Стандартный `java.nio.file.WatchService` работает в отдельном daemon-потоке. Проблема отслеживания вложенных папок решена через ручной рекурсивный обход `Files.walkFileTree`. Чтобы чтение файлов не блокировало прослушивание системных событий, тяжелая работа (I/O и парсинг текста) делегируется пулу потоков `ExecutorService`.
* **Расширяемая токенизация (Pattern Strategy):**
  Логика разбиения текста вынесена в отдельный интерфейс `Tokenizer`. По умолчанию используется `SimpleTokenizer` на базе Java Stream API (разбиение по пунктуации, приведение к нижнему регистру, фильтрация пустых строк), но пользователь библиотеки может легко реализовать собственный лексер со стеммингом или удалением стоп-слов.

## Как запустить

### Предварительные требования
* JDK 17 или выше (использован современный синтаксис `Files.readString` и `Set.copyOf`).
* Установленный Gradle (или можно использовать встроенный `./gradlew`).

### Сборка и запуск тестов
Выполните из корня проекта:
```bash
# Сборка всех модулей и прогон unit-тестов
./gradlew build
